# SmartRoute Port Sync — Field Report

A cross-workspace live dashboard showing the JoinOurBuild / SmartRoute (JOB) ↔ SabineMR / new-smartroute (PER) port ledger.

- **224 tickets** ported across two Linear workspaces, organized by their 16 epic parents
- Live status pills pull directly from the Linear GraphQL API for each workspace
- Cached 5 min per request (Next.js ISR), manual refresh button in header
- Deployed as a standalone Next.js 15 app on Vercel

## Design direction — "Field Report"

Editorial data-journalism aesthetic. Fraunces (serif display) + IBM Plex Sans (body) + IBM Plex Mono (identifiers), warm off-white paper background, muted status palette, hair-rule dividers. Reads like a printed report you're scanning at a desk. Not AI-punk.

## Getting started

```bash
pnpm install
cp .env.example .env.local
# Add your Linear personal API keys, one per workspace
pnpm dev
# Opens on http://localhost:3200
```

### Linear API keys

Generate at `https://linear.app/[workspace]/settings/api` in each workspace:

- `LINEAR_JOB_API_KEY` — from the **JoinOurBuild** workspace (scoped to read the SmartRoute project)
- `LINEAR_PER_API_KEY` — from the **SabineMR** workspace (scoped to read the new-smartroute project)

Both are personal API keys with read scope. Store in Vercel project env vars for deployment.

## Data flow

1. `src/data/port-map.json` — the canonical 224-entry JOB→PER mapping (baked at build)
2. `src/data/summaries.json` — 1-3 sentence summaries + type classifications (baked at build)
3. `src/lib/linear.ts` — fetches current status/assignee for every ticket from both workspaces at request time
4. `src/lib/join.ts` — joins baked data + live data into a hierarchical structure grouped by epic
5. `src/app/page.tsx` — the Server Component that renders the field report

## Deployment

- Import repo into Vercel from GitHub
- Add both `LINEAR_*_API_KEY` env vars in the Vercel project settings
- Deploy — the public URL is shareable
